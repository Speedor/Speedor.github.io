Docker使用指南

       虽然在大学很早就听说了Docker这个东东而且自己也并不是做Docker运维的，但是求知欲的驱使难免会想搞清楚这个东西到底是什么，怎么用，用在哪。由于工作也或多或少会用到Docker，所以这次抽了点时间对Docker的主要知识点进行了整理，也是自己学习过程中理论和实践的理解，一是对自己这段时间Docker的总结，二是希望大家对我的理解和看法有不同之处请指出来相互学习，来喷我啊～～

Docker 简介

       老样子啦，还是以官方文档为准，英文官方在这儿，中文的在这儿，下边只是个人学习过程中觉得相对重要的点，仅供参考，来喷我～

Docker 两个主要用途：

  • Docker: 开源的容器虚拟化平台
  • Docker Hub: 用于分享、管理 Docker 容器的 Docker SaaS 平台：Docker HUB : Docker镜像首页，包括官方镜像和其它公开镜像，因为国情的原因，国内下载 Docker HUB 官方的相关镜像比较慢，可以使用 docker.cn 镜像，镜像保持和官方一致，关键是速度块，推荐使用

Docker 内部构建，需要理解以下三种部件：

  • Docker 镜像 - Docker images
  • Docker 仓库 - Docker registeries
  • Docker 容器 - Docker containers

       Docker 使用客户端-服务器 (C/S) 架构模式。Docker 客户端会与 Docker 守护进程进行通信。Docker 守护进程会处理复杂繁重的任务，例如建立、运行、发布你的 Docker 容器。Docker 客户端和守护进程可以运行在同一个系统上，当然你也可以使用 Docker 客户端去连接一个远程的 Docker 守护进程。Docker 客户端和守护进程之间通过 socket 或者 RESTful API 进行通信。Docker 守护进程运行在一台主机上。用户并不直接和守护进程进行交互，而是通过 Docker 客户端间接和其通信。

Docker 通信

       现在我们创建好了容器，但是我们要怎么让容器间互相通信呢，毕竟单个容器不能发挥他的强大之处，实际并不难因为在Docker1.9后的network:1.9之后，在Docker中network从实验特性转为正式特性发布，这让我们的使用更加稳定简便高效。

Docker服务安装成功后默认有四个网络模式：

       docker run 创建 Docker 容器时，可以用 --net 选项指定容器的网络模式。

  • bridge 模式，容器默认设置。此模式会为每一个容器分配 Network、设置 IP 等，并将一个主机上的 Docker 容器连接到一个虚拟网桥上。
  • host 模式，使用 --net=host 指定。容器不会虚拟出自己的网卡，配置自己的 IP 等，而是使用宿主机的 IP 和端口。
  • container 模式，使用 --net=container:NAMEorID 指定。指定新创建的容器和已经存在的一个容器共享一个 Network，而不是和宿主机共享。
  • none 模式，使用 --net=none 指定。Docker容器不会进行任何网络配置。也就是说，此容器没有网卡、IP、路由等信息。需要我们为容器手动添加网卡、配置 IP 等。

       是不是感觉有点像VMware和VirtualBox的网桥模式，这里这可对比着来看，但是他们还是有些区别的不能一概而论，毕竟是两种不同的虚拟方式。我们可以通过ifconfig查看主机IP，配置容器IP并设置在同一局域网内，然后进行互ping，因为网络配置主要以实际需求为主，我们还是要从实际出发，根据业务逻辑进行配置，所以这里我就不过多描述了，因为工具的使用还是要靠查看文档和动手实践相结合的方式进行学习，相信在一段时间的接触后，Docker会从知识的恐慌区进入学习区最终到达舒适区，成为我们工作学习中得心应手的工具。
