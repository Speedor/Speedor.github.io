深度学习之TensorFlow初探

       跟着一位大牛接触了一段时间深度学习很感谢他带我入门，当然啦还是要膜拜一下“花书”，后来接触过程中发现这个东西还是有点意思的，尤其是看到TensorBoard这个神奇的可视化工具后感觉像是Google为我们开了一扇新的大门。其实Google对于TensorFlow的版本迭代更新差不多两个多月就有一个更新，可见Google对TensorFlow的重视程度，从TPU的推进和TensorFlow的更新就可以看出Google的生态圈正在逐步完善，应该离形成闭环也不远了，可以想像Google的生态圈将会更加强大和稳健。

TensorFlow简介

       TensorFlow实际上Tensor（张量）和Flow（流）的合成词，我们可以先理解为数据流动（我是这么理解的），大家有不同的理解方式可以提出来一起学习，TensorFlow的中文社区做的挺好，可以做为入门参考，毕竟它更新太快我们还是尽量参考官网最好，另外极客学院也对其做了一次中文翻译，大家可以根据自己不同的情况进行学习。

       虽然Google开源了TensorFlow，在简化我们工作的同时本质上还是要用扎实的理论知识做支撑，我们还是要做好基础然后再用TensorFlow做一些实践，两者相结合进行学习，当然这是我的学习过程，大学觉得有好的可以提出来一起学习。

       TensorFlow是一个采用数据流图（data flow graphs），用于数值计算的开源软件库。节点（Nodes）在图中表示数学操作，图中的线（edges）则表示在节点间相互联系的多维数据数组，即张量（tensor）。它灵活的架构让你可以在多种平台上展开计算，例如台式计算机中的一个或多个CPU（或GPU），服务器，移动设备等等。

       上边图就是数据流图（Data Flow Graph），我们就可以通过编程作图并控制数据流动，数据流图用“结点”（nodes）和“线”(edges)的有向图来描述数学计算。“节点” 一般用来表示施加的数学操作，但也可以表示数据输入（feed in）的起点/输出（push out）的终点，或者是读取/写入持久变量（persistent variable）的终点。“线”表示“节点”之间的输入/输出关系。这些数据“线”可以输运“size可动态调整”的多维数据数组，即“张量”（tensor）。张量从图中流过的直观图像是这个工具取名为“Tensorflow”的原因。一旦输入端的所有张量准备好，节点将被分配到各种计算设备完成异步并行地执行运算。

TensorFlow编程

       操作时我们要记住TensorFlow两个概念：图(Graph)和会话(Session)

  • 使用图 (graph) 来表示计算任务.
  • 在被称之为 会话 (Session) 的上下文 (context) 中执行图.
  • 使用 tensor 表示数据.
  • 通过 变量 (Variable) 维护状态.
  • 使用 feed 和 fetch 可以为任意的操作(arbitrary operation) 赋值或者从其中获取数据.

Feed

       feed 使用一个 tensor 值临时替换一个操作的输出结果. 你可以提供 feed 数据作为 run() 调用的参数. feed 只在调用它的方法内有效, 方法结束, feed 就会消失. 最常见的用例是将某些特殊的操作指定为 "feed" 操作, 标记的方法是使用 tf.placeholder() 为这些操作创建占位符.
